## Pull requests

For every pull request created from this repository:

- Use a descriptive title in the format `<type>: <clear outcome>`, for example `feat: add role-scoped operational reports`.
- Complete every relevant section of `.github/PULL_REQUEST_TEMPLATE.md` with specific, developed prose.
- Never leave placeholder text in the final pull request description.
- Include the affected roles and views, data or migration impact, deployment configuration, security considerations, and the exact validation commands and results.
